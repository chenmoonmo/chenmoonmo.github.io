    前后端通信方式

    AJAX（Asynchronous JavaScript and XML）

    AJAX是浏览器提供的一套可以实现页面无刷新更新数据的方法，可以提高用户的浏览体验。

    应用场景：

    1. 页面上拉加载更多
    2. 列表数据无刷新分页
    3. 表单离开焦点数据验证
    4. 搜索框提示文字下拉列表

    AJAX实现步骤

    请求参数传递

    • get: url?key1=value1&key2=value2… 遍历拼接字符串

    • post: xhr.sed(params) 传递JSON字符串参数 JSON.stringify() 将对象转为JSON字符串

    请求头设置

    • get: xhr.setRequestHeader(‘content-Type’, ‘application/x-www-form-urlencoded’)
    • post: xor.setRequestHeader(‘content-Type’, ‘application/json’)

    get请求和传统表单都不能提交JSON格式

    AJAX状态码(使用xhr.readystate获取 写在send方法之前)

    0: 请求为初始化

    1: 请求已经建立，但还没有发送

    2: 请求已经发送

    3: 请求正在处理中

    4: 响应已经完成，可以获取并使用服务器响应的数据了

    oneadystatechange 事件：当AJAX状态码变化时自动触发

    最好在onreadystatechange事件中判断readystate为4时，再接受xhr.responseText

    fetch

    fetch是一种基于promise实现的功能更加强大，更灵活，更简单的数据获取方式，可看作是xhr的升级版，ES6原生方法。

    fetch使用方法

    fetch请求参数

    • 请求方法 method(string): ‘get’获取数据, ‘post’传输数据, ‘put’修改数据, ‘delete’删除数据
    • 请求参数 body(string)
    • 请求头 headers(object)
    • 是否跨域 mode: ‘cors’

    post和put

    fetch响应结果

    • text() 将返回体处理成字符串类型
    • josn() 返回结果和JSON.parse(response)一样的对象类型

    axios

    axios是基于promise用于浏览器和node.js的HTTP客户端，是第三方库。

    特征：

    • 支持浏览器和node.js
    • 支持promise
    • 能拦截请求和响应
    • 自动转换为JSON数据

    浏览器用法

    get和delete传参 服务器端
    axios.get(‘/path?key=value’) req.query
    axios.get(‘/path/value’) App.get(‘path?:参数名’)
    通过params选项传参数 req.query

    params传参：

    Axis.get(‘/path’,{

    ​ params: {

    ​ key: value

    ​ }

    })

    post和put传参

    • 通过选项传递参数
    • 通过URLSearchParams传参(application/x-www-form-urlencoded)

    axios的响应结果

    主要属性：

    • data 实际响应的数据
    • headers 响应头信息
    • status 服务器状态码
    • statusText 响应状态信息

    axios全局配置

    • 超时时间 axios.defaults.timeout = 3000
    • 默认地址 axios.defaults.baseURl = ‘http://…’
    • 设置请求头 axios.header[‘…’] = ‘…’

    axios拦截器

    • 请求拦截器： 在请求发出前发出一些信息
    • 响应拦截器: 在服务器数据响应给客户端之前做一些加工处理

    同源政策

    同源：两个页面拥有相同的协议、域名和端口

    同源政策：出于保护用户信息的安全，防止网站窃取数据，不同源的页面之间不能互相请求信息，</img>和<script>标签例外。
